# YouTube Shorts 업로드 최적화 스킬

> **업로드 전 이 체크리스트를 반드시 완수한다. 하나라도 빠지면 업로드 금지.**

---

## 1. 제목 (Title) — 40자 이내

### 규칙
- **40자 이내** — 모바일에서 잘리지 않는 길이
- **핵심 키워드 앞 1/3** — 검색 노출 + 알고리즘 인식
- **선언형 > 질문형** — "~한 남자" > "~했을까?"
- **숫자 포함** — "7명을 죽이고", "17세 소녀가" → 시선 끔
- **호기심 갭** — 결과를 암시하되 완전히 공개하지 않음

### 범죄 쇼츠 제목 공식
```
[충격적 사실/숫자] + [핵심 키워드] + [호기심 갭]
```
예시:
- "가족을 죽이라 했더니 정말로 죽였다" (18자)
- "17세 소녀가 뛰어나온 잠긴 방의 비밀" (17자)
- "7명을 죽이고도 증거가 없었다" (14자)

### 금지
- 50자 이상 제목
- 낚시성 제목 (내용과 불일치)
- 모든 글자 대문자/특수문자 남용
- "충격", "경악", "소름" 같은 과장 클리셰 단독 사용

---

## 2. 설명 (Description) — 150자 이상

### 구조
```
[1줄] 시리즈명 — 주제
[1줄] Part N: 부제

[2-3줄] 핵심 내용 요약 (검색 키워드 자연 삽입)

📌 Part 1 ▶ [링크]
📌 Part 2 ▶ [링크]
📌 Part 3 ▶ [링크]

🔔 구독하고 다음 사건을 기다려주세요

#Shorts #범죄 #실화 #사건기록부 #[사건키워드]
```

### 핵심
- **첫 2줄**이 "더보기" 전에 보임 → 가장 중요한 내용
- **시리즈 링크** 반드시 포함 → 다른 편 유도
- **구독 CTA** 명시적으로
- **해시태그**는 설명 맨 아래에 (제목에 넣지 않음)

---

## 3. 해시태그 — 3-5개, 설명에 배치

### 필수 구성
```
#Shorts             ← 쇼츠 피드 노출 신호 (필수)
#범죄               ← 니치 태그 1
#실화               ← 니치 태그 2
#사건기록부          ← 채널 브랜드 태그
#[사건키워드]        ← 영상 고유 태그 (기타큐슈, 연쇄살인 등)
```

### 규칙
- **3-5개** — 그 이상은 스팸 판정 위험
- **설명에 배치** — 첫 3개가 제목 위에 자동 표시
- 영상 내용과 **직접 관련** 있는 태그만
- 트렌딩 해시태그 1개 선택적 추가 가능

---

## 4. 썸네일 — 커스텀 1080×1920

### 규격
- **크기**: 1080×1920px (9:16)
- **텍스트**: 굵은 글씨 2-3단어 (제목 핵심)
- **대비**: 어두운 배경 + 밝은 텍스트 (노랑 #FFD000 / 빨강 #FF2200)
- **얼굴/인물**: 가능하면 포함 (CTR 향상)

### 제작 방법
- 영상의 가장 인상적인 프레임 추출
- 또는 별도 그래픽 제작 (Remotion 첫 프레임 활용)
- 작은 화면(홈피드)에서 가독성 테스트 필수

---

## 5. 업로드 시간대 (한국 기준)

| 요일 | 추천 시간 | 이유 |
|------|----------|------|
| 화~금 | **17:30** | 퇴근 피크(18-20시) 1시간 전 |
| 토~일 | **10:00** | 주말 피크(10-12시) 직전 |
| 월요일 | 피하기 | 월요일 시청률 최저 |

### 시리즈 업로드 간격
- 시리즈물: **1-2일 간격** 연속 업로드
- Part 1 → (1일) → Part 2 → (1일) → Part 3
- 같은 시간대 고정 → 시청자 습관 형성

---

## 6. 고정 댓글 — 참여도 부스터

### 공식
```
[영상 핵심 사실 1줄] + [찬반 질문 또는 시청자 의견 요청]
```

### 채널별 고정 댓글 템플릿

**incidents (사건기록부)**:
```
마쓰나가는 자기 손에 피를 한 방울도 묻히지 않았습니다.
여러분은 이런 사람을 옆에서 알아볼 수 있다고 생각하시나요?

📌 Part 1부터 보기 ▶ [링크]
```

### 효과
- 찬반 질문 → 댓글 수 폭증
- 댓글 읽기 위해 체류 → 시청시간 증가
- 참여도 높은 영상 = 알고리즘 추천 강화

---

## 7. 재생목록 & 시리즈 연결

### 시리즈물 필수 조치
1. **재생목록 생성**: "[사건명] 시리즈" 재생목록
2. **순서 지정**: Part 1 → 2 → 3 순서
3. **각 편 설명에 다른 편 링크**: Part 2 설명에 Part 1, 3 링크
4. **채널 홈 섹션**: 시리즈 재생목록을 채널 홈에 배치

## 8. 알고리즘 핵심 신호 (우선순위)

| 순위 | 신호 | 우리 영상에서 |
|------|------|-------------|
| **1** | 스와이프 이탈률 (3초) | Hook "오늘의 기록" + 충격 사실 |
| **2** | 완주율 | 100초 적정, 클리프행어로 끝까지 유도 |
| **3** | 참여율 | 고정 댓글 질문으로 댓글 유도 |
| **4** | 리플레이율 | 충격 전개 → 다시 보기 유도 |

## 9. 업로드 전 최종 체크리스트

```
□ 제목: 40자 이내, 키워드 앞쪽, 호기심 유발
□ 설명: 150자+, 시리즈 링크, 구독 CTA, 키워드 포함
□ 해시태그: #Shorts #범죄 #실화 #사건기록부 #[사건명] (설명에)
□ 썸네일: 커스텀 1080×1920, 굵은 텍스트, 작은 화면 가독성
□ 재생목록: 시리즈 재생목록에 추가
□ 고정 댓글: 찬반 질문 작성 (업로드 직후 즉시 고정)
□ 업로드 시간: 평일 17:30 / 주말 10:00
□ 시리즈 간격: 1-2일 간격 순차 업로드
□ 영상 품질: 1080×1920, h264, 60초 이상은 쇼츠 아님 주의
□ 자막 최종 확인: 오타/오전사 0건
```

- **공개 범위**: 공개

## 11. 48시간 알고리즘 체크리스트 (Phase 45)

> **업로드 직전 9항 전부 통과해야 게시 허용.** 미충족 시 WARNING 로그 + 수동 확인 유도.

| # | 항목 | 검증 방법 | 자동/수동 |
|---|------|----------|----------|
| 1 | 제목이 "문제 중심" 포지셔닝 | FAIL 패턴 미매칭 (title_positioning.py) | 자동 (ins-blueprint INS-BLP-02) |
| 2 | 시의성 키워드 옵션 | seasonal_keywords 적용 여부 | 수동 확인 |
| 3 | 초반 3초 훅에 메인 키워드 음성 | hook narration에 main_keyword 포함 | 자동 (ins-structure INS-STR-03) |
| 4 | 고정댓글 키워드 타이핑 유도 | pinned_comment에 engagement_question 포함 | 자동 생성 |
| 5 | RapidTags 5개 태그 | tags >= 5 (rapidtags_fetch.py + fallback) | 자동 |
| 6 | 설명란 첫 2문장에 핵심 키워드 | description 첫 2문장에 main_keyword 포함 | 자동 (upload_metadata_build.py) |
| 7 | 채널별 peak_upload_hour | publish_at = channel_upload_profile.peak_upload_hour_kst | 자동 |
| 8 | 엔딩 스크린 | 쇼츠 N/A (YouTube 제한), 롱폼만 해당 | N/A |
| 9 | 시리즈 재생목록 설명란 | playlist description 자동 채움 (series_info 존재 시) | 자동 |

### 검증 흐름
1. 영상 제작 완료 후 `validate_48h_checklist()` 호출
2. 에러 0건 → 업로드 진행
3. 에러 있음 → WARNING 로그 출력 + 수동 확인 유도 (CRITICAL 차단 아님)

### 원전
- `config/benchmark-48h-playbook.md` — 자생법 벤치마크 전체
- `config/channel_upload_profile.json` — 채널별 프로파일
