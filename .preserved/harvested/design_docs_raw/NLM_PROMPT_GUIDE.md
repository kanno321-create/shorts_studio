## 📌 배경 — 왜 이 규칙이 확정됐나

### 실측 증거

**Phase 47 실패 (2026년 초)**:
- 12 섹션 강제 / 섹션별 글자수 / 감정 배정 / 금지어 리스트 / JSON 스키마 / 자체점검 5단계를 **전부 NLM 프롬프트에 박음**
- 결과: E2E 2편 (incidents + humor) **둘 다 "재미없음"으로 대표님 판정 FAIL**
- 기술적 PASS (검사관 OK), **재미 FAIL** (시청자 관점 실패)

**세션 76 대표님 발견 (2026-04-18)**:
- "강제하는것보다, 내가 원하는 방향의 대본이되 몇분이내로해달라고 하니까 굉장히 좋은 품질이 나오더라"
- 보완: "**그대신 원하는 내용을 잔뜩 쓰지**"

### 왜 형식 강제가 품질을 죽이는가

1. **채워넣기 모드 전환**: NLM이 형식 컴플라이언스에 에너지 쓰면 스토리 에너지가 고갈됨
2. **창의 자유 박탈**: NLM은 "좋은 이야기꾼"인데 체크리스트를 주면 이야기가 죽음
3. **쿼리 폭증**: 강제 항목 많을수록 follow-up으로 "X가 빠졌다"/"Y 글자수 안 맞다" 재요청 → 편당 2500+ 쿼리 소비 → Ultra 한도로도 2편이면 고갈

## ❌ DON'T — 이런 건 절대 쓰지 마라

### 형식 강제 (Negative Constraint — 재미 압살)

- ❌ "12 섹션으로 나눠"
- ❌ "Hook 20자 이내, Body 각 60자, CTA 50자"
- ❌ "JSON 스키마 정확히 맞춰: {hook, body[], cta}"
- ❌ "자체점검 5단계 후 출력"
- ❌ "금지어 리스트: 믿으시겠습니까/놀라운 사실/상상을 초월/충격/설마" (20개+)
- ❌ "감정을 섹션별로 배정: section 1=normal, 2=tonedown, 3=whisper..."
- ❌ "섹션별 emotion + silence_after_ms + style_tag + character 전부 채워"
- ❌ "이전 대화 무시, 이 프롬프트만 기준" (리셋 명령도 불필요)

### 왜 이게 재미를 죽이는가

```
[강제 프롬프트 예시 — FAIL 패턴]
"D.B.쿠퍼 사건, 12 섹션 JSON:
- Hook 20자, Body 1~10 각 60자, CTA 50자
- 감정 배정: normal/tonedown/whisper/toneup 중
- 자체점검 5단계
- 금지어 20개 회피
- 한국어 번역체 패턴 6개 금지
- 숨결 연결재 3~4문장에 1회 필수
- 체언종결 2개 연속 금지"

→ NLM이 "형식 맞추기" 모드. 이야기 에너지 0.
→ 검사관은 PASS할 수 있음. 시청자는 재미없음.
```

## ⏱️ 시간 제약은 **유일한** 수치

- "110초 이내" / "90초" / "120초 이내" — 한 개만
- 이걸로 분량 자동 조절됨
- 섹션 수·글자수·구조는 **모델 자율**에 맡김

## 🌟 좋은 예시 — 전체 완성본

```
D.B.쿠퍼 사건 다뤄줘.

분위기:
- 1971년 가을밤 오리건의 공기. 조용하고 차갑게.
- 탐정이 사건 파일을 오래 들여다본 사람의 피로감으로 이야기함.
- 즌다(조수)는 시청자가 궁금해할 지점을 짚어주는 역할.
  "왜 뛰어내렸을까요?" 같은 질문 한두 개 필수.

살려주고 싶은 것:
- FBI가 45년 매달리고 2016년 공식 포기했다는 사실의 무게
- 돈뭉치 일부가 1980년 강변에서 발견된 장면 (한 cut 정도)
- 쿠퍼라는 이름이 가짜였다는 반전을 마지막 15초쯤

피하고 싶은 것:
- "놀라운 사실이 밝혀졌습니다" 류 AI 클리셰
- 한국어 번역체 느낌 (일본어 영상이면 특히)
- 감정을 대사로 직접 설명 — 장면/행동으로 보여줘

분량: 110초 이내.

나머지(섹션 개수·글자수·구조)는 네가 이 이야기에 가장 어울린다고 판단한 대로.
```

**이 프롬프트 구성**:
- 분위기·주안점·살릴 것·피할 것 → **10줄 넘게** 풍부
- 수치 → "110초 이내" 하나만
- 구조 지시 → "네가 판단한 대로" (모델 자율)

## 🚨 나쁜 예시 — 이러면 재미 FAIL

```
D.B.쿠퍼 사건 쇼츠 대본 작성.

요구사항:
1. 섹션 수: 정확히 12개 (hook 1 + body 10 + cta 1)
2. Hook: 20자 이내, 체언 종결 금지, 의문문 권장
3. Body 각 섹션: 60자 ±5자
4. CTA: 50자 이내
5. 감정 배정: normal/tonedown/whisper/toneup 중 선택, 연속 3개 이상 같으면 안됨
6. silence_after_ms: 각 섹션 100~1000
7. 금지어: 믿으시겠습니까/놀라운 사실/상상을 초월/충격의 진실/설마의 전개/진짜입니까/충격/반전/공포/미스터리
8. 번역체 6패턴 회피: ~라고 할 수 있다/~인 것이다/~라고 여겨진다/~되어지다/~에 있어서/~에 대해서
9. 체언 종결 2개 연속 금지
10. 숨결 연결재 ("...군요", "그런데", "이건...") 3~4문장에 1회
11. 출력 JSON: {hook: {text, emotion}, body: [{text, emotion, silence_after_ms}], cta: {text}}
12. 자체점검: 출력 후 다음 5가지 검증 후 답변
    (a) 섹션 수 12개인가
    (b) 각 글자수 허용 범위인가
    (c) 금지어 0건인가
    (d) 번역체 6패턴 ≤ 2회인가
    (e) 체언 종결 연속 없는가
13. 이전 대화 무시, 이 프롬프트만 기준으로 작성
```

→ NLM이 **이야기 에너지 0, 컴플라이언스 100**로 전환.
→ Phase 47 E2E 2편이 정확히 이 패턴으로 "재미없음" FAIL.

## 💾 자체점검·금지어 검사 — NLM 안에서 하지 말 것

### 잘못된 방식
```
❌ "출력 후 자체점검해: 금지어 없는지, 섹션 수 맞는지, 글자수 맞는지..."
```
→ NLM이 자체점검에 에너지 낭비 + 쿼리 폭증

### 올바른 방식
```
✅ 1. NLM에서 "방향 + 시간"만으로 대본 받기
   2. 나베랄이 받은 뒤 오프라인 처리:
      - Python grep으로 금지어 확인
      - 글자수 측정 스크립트
      - 번역체 6패턴 grep
      - 체언 종결 연속 감지
   3. 문제 발견 시 → NLM에 **방향 문장 재작성**으로 재요청
      ("번역체가 섞였어. 더 자연스러운 일본어로 다시")
```
