## 1. 아키텍처 — 멀티 에이전트 오케스트레이션

```
순차 파이프라인 금지. 아래 구조로만 영상을 제작한다.

디렉터 에이전트
  │  전체 설계도 작성 (도입부/본론/결론 + 장면별 영상 지시)
  │
  ├─ 대본 에이전트: 설계도 기반 타임라인 대본
  ├─ 영상수집 에이전트: 커뮤니티/SNS에서 실제 관련 영상 수집
  ├─ TTS 에이전트: 지정된 보이스로 음성 생성
  ├─ 자막 에이전트: 단어 단위 싱크 자막
  └─ 편집자 에이전트: 모든 재료 받아 최종 조립 + 싱크 맞춤
        │
디렉터 에이전트: 최종 검수 → 합격/불합격 판정
```

### 디렉터 설계도 필수 항목
- 도입부: 훅 문장 + 시청자 호기심 유발
- 본론: 상황 설명 + 각 장면에 "어떤 영상이 필요한지" 구체 지시
- 결론: 시청자 참여 유도 (댓글, 의견 요청)
- 예시: "너희들 그거 알어? → 상황설명 → 너희는 어떻게 생각해?"

### 소재 발굴 (방식 A + B 모두 가능)
- **방식 A**: 주제 먼저 → 관련 영상/게시물 수집
- **방식 B**: 커뮤니티에서 이슈 영상 발견 → 그 영상으로 대본 작성
- 영상이 없어도 이슈된 게시물 내용만으로 대본 작성 가능

## 2. 영상 소스 — Pexels/Pixabay 스톡 단독 사용 금지

### 반드시 사용할 소스
| 우선순위 | 소스 | 설명 |
|----------|------|------|
| 1 | **실제 관련 영상** | 커뮤니티/SNS에서 해당 사건의 원본 영상 |
| 2 | **관련 인물 영상/사진** | 트럼프 뉴스 → 트럼프 영상, 정치인 → 해당 인물 |
| 3 | **커뮤니티 캡처** | 게시글 스크린샷, 댓글 캡처, 기사 화면 |
| 4 | **AI 생성 (Veo 3.1 Lite)** | 사진에서 짧은 영상 생성, 실제 영상이 없을 때 |
| 5 | **일러스트** | 만화풍 캐릭터, 상황 묘사 그림 |
| 6 | 스톡 (최후 수단) | 위 5가지로 해결 불가할 때만 보조로 사용 |

### 수집처
Reddit, Twitter/X, TikTok, Instagram, 웨이보, 디시인사이드, FM코리아, 루리웹, 2ch/5ch, 각국 커뮤니티

**YouTube (조건부 — 2026-04-17 정책 개정)**:
- Creative Commons (CC-BY 4.0) 라이선스 영상에 한해 다운로드 허용
- 호출: `web_source.search_youtube_cc(query)` 또는 자동 분기 (`download_content` 안에서 YouTube URL 만나면 라이선스 검증 후 CC만 통과)
- 비-CC 영상은 `download_youtube_cc()`가 `status="blocked"` 반환 (예외 raise 없음)
- 출처/라이선스 명시는 영상 설명에 자동 기록 권장

### 금지
- Pexels/Pixabay 스톡만으로 영상 전체를 구성하는 것
- 내용과 무관한 "분위기" 영상 사용 (트럼프 뉴스에 도시 풍경 등)
- ~~유튜브 영상 다운로드~~ → **YouTube 비-CC 영상 다운로드** (CC-BY는 허용으로 정책 변경, 2026-04-17)

## 3. 자막 — 단어 단위 빠른 전환

### 현재 (금지)
```
"고객이 이층으로 던져달라 했더니 쿠팡맨이 진짜 이층으로 던졌습니다"
→ 한 문장 전체가 한번에 나옴 (읽기 불편)
```

### 올바른 방식
```
"고객이" → "이층으로" → "던져달라 했더니" → "쿠팡맨이" → "진짜" → "이층으로 던졌습니다"
→ 내레이션 타이밍에 맞춰 5-8글자씩 빠르게 전환
```

### 규칙
- faster-whisper 단어 단위 타이밍 사용
- 한 화면에 최대 8글자 (1줄)
- 내레이션 속도와 정확히 싱크
- 굵은 폰트 + 테두리 + 그림자

- 나머지 텍스트: 흰색(#FFFFFF)
- 배경: #0a0a0a + 미세 그리드 패턴 (고급감)

### 자막 (영상 센터 영역 내 오버레이)

- spring/scale 애니메이션 금지 — 색상 변경만 허용

## 6.5. 영상 길이 — 보이스별 분량 매트릭스 (세션 40 재구성)

> **세션 40 측정 결과 기반**: Morgan (incidents) 는 영어 보이스의 한국어 발음이라 자연 속도가 약 6.9자/s 로 빠르다. 한국어 차분 보이스 (4.2자/s) 와 같은 매트릭스에 묶으면 같은 글자수에서 duration 이 약 60% 짧다. 그래서 매트릭스는 **톤이 아니라 보이스별 실측 속도** 로 분리한다.

### 보이스 그룹 분류 (자연 속도 실측)

| 그룹 | 보이스 / 채널 | 자연 속도 (tempo 1.0) | 실측 (tempo 0.88) |
|------|----------|---------------------|-------------------|
| **느린 톤** | (한국어 차분 보이스, 미정) | ≈ 4.2 자/s | ≈ 4.8 자/s |
| **중간 톤** | **Morgan** (incidents) | ≈ 7.9 자/s | ≈ 6.9 자/s |
| **빠른 톤** | 박창수 (humor), 필재 (sseoltube), 카밀라 (trend), 소진/mc콩 (politics) | ≈ 8.5 자/s | n/a |

> 톤 = 청각 인상 (차분 vs 빠름), 자연 속도 = 글자수 × 시간 비율. 두 값은 일치하지 않을 수 있다 — Morgan 은 톤은 차분하지만 실제 글자/초는 중간 정도. 매트릭스는 **글자수 기준** 이라 자연 속도를 따른다.

### 분량 매트릭스 (자동 pause 주입 0.35s/0.2s 적용 후 최종 duration 기준)

**중간 톤 — Morgan (incidents)**

| 유형 | duration | 본문 글자수 권장 | 비고 |
|------|----------|----------------|------|
| 단편 | 50-65 s | 약 **350자** (320-400) | |
| 시리즈 각 편 | 90-130 s | 약 **620자** (550-720) | |
| **상한** | **140 s** | **750자** | 시청 한계 |

**느린 톤 — 한국어 차분 보이스 (incidents-jp / health / sseoltube)**

> ⚠️ 이 표는 한국어 차분 보이스 (예: 태백) 도입 후에만 적용. 현재 incidents 는 Morgan (중간 톤) 사용.

| 유형 | duration | 본문 글자수 권장 |
|------|----------|----------------|
| 단편 | 50-65 s | 약 **220자** (200-260) |
| 시리즈 각 편 | 100-140 s | 약 **520자** (450-580) |
| **상한** | **140 s** | **580자** |

**빠른 톤 (humor / politics / trend)**

| 유형 | duration | 본문 글자수 권장 |
|------|----------|----------------|
| 단편 | 50-60 s | 약 **460자** (425-510) |
| 시리즈 각 편 | 80-110 s | 약 **800자** (680-935) |
| **상한** | **120 s** | **1020자** |

> **차분 톤 시리즈는 130초 까지 허용** — 탐정 독백 / 지식 전달은 호흡과 pause 가 본질이므로 빠른 톤(120초 상한)보다 10초 더 긴 분량 허용. 이 차이는 임의 완화가 아니라 톤 자체의 본질에서 나옴.

### 시리즈물 규칙
- 각 편 끝에 **클리프행어** 필수 (다음 편으로 유도)
- 채널 페이지에서 시리즈를 찾아보게 만드는 것이 목적
- 편 간 이전 내용 요약 없이 바로 본론 진입
- **Part 1**: 인삿말 + 사건 도입 + 클리프행어
- **Part 2~N-1**: 인삿말 + 본론 + 클리프행어
- **Part N (마지막)**: 인삿말 + 본론 + 끝맺음말

### 나레이션 감정 규칙

- 전체 `normal` 감정으로 읽기 **금지** — 섹션/문장별 감정 변화 필수, 최소 3개 감정 사용
- **사용 가능 감정**: `normal`, `happy`, `angry`, `whisper`, `toneup`, `tonedown`
- **`sad` 절대 금지** — Typecast 볼륨 급감 버그 (메모리 `feedback_sad_volume_low.md`). 슬픈 톤이 필요하면 `tonedown` + volume 보정으로 대체
- 극적 순간 앞뒤로 **무음 간격** 0.5~1.0 s (`silence_after_ms` 필드)
- 숨결 연결재 ("...군요", "그런데", "이건...") 3~4문장에 1회
- **마침표/쉼표 pause 는 `tts_generate.py` 자동 주입** — Scripter 는 본문 작성에만 집중. `<break />` 수동 삽입 불필요 (보이스 preset 에 `auto_punctuation_pause: true` 가 켜진 채널 한정, 메모리 / FAILURES 참조)

### 문체 규칙 — 서사체 필수 (FAIL-SCR-001 영구 등록)

**모든 채널 공통, 위반 시 Scripter Step 6 자동 FAIL**:

- ❌ **체언(명사) 종결 단문 2개 이상 연달아 금지** — "열일곱 살. 팔에 전기 화상." 같은 개조식 나열은 AI 스럽고 딱딱함
- ✅ **모든 문장은 서술어(동사/형용사) 종결**
- ✅ **정보 조각은 관형절로 묶어라** — "팔과 다리에 전기 화상을 입은 열일곱 살 소녀가"
- ✅ **연결 형태소** (-고, -며, -어서, -는데, -ㄴ 채, -자) 로 문장 사이 연결고리 만들기
- ✅ **incidents/politics 형사톤 형용사 허용** — 급하게, 기이하게, 섬뜩하게, 처절하게, 묵묵히 (메모리 `feedback_channel_adjective_style.md`)
- ✅ **평균 문장 길이 12자 이상** — 그 미만이면 개조식 의심, Scripter Step 6 경고

각 채널별 BAD/GOOD 원문 예시는 `.claude/skills/channel-{name}/SKILL.md` 의 "문체 규칙" 섹션 참조.

### 채널 시그니처 캐치프레이즈 (FAIL-SCR-002 영구 등록)

각 채널의 인삿말은 **Hook section[0]** 에, 끝맺음말은 **CTA 마지막 section** 에 반드시 포함. 위치 오류 (body 에 옮기기, 변형) 도 FAIL.

| 채널 | 인삿말 (Hook 의무) | 끝맺음말 (CTA 의무) |
|------|-------------------|---------------------|
| incidents | `"오늘의 기록."` | `"저는 이 사건을 아직 놓지 않았습니다."` |
| incidents-jp | `channel-incidents-jp/SKILL.md` 의 캐치프레이즈 섹션 참조 | 동상 |
| humor / politics / trend / sseoltube / health | 각 채널 SKILL.md 캐치프레이즈 섹션 참조 (없으면 대표님 에스컬레이션) | 동상 |

> **시리즈 Part N (N < M)**: 끝맺음말 + 연결형(`"다음 기록에서, 이어집니다"`) **둘 다** 사용
> **시리즈 마지막 편**: 끝맺음말만 사용

## 7. 제출 전 체크리스트 (하나라도 NO면 제출 금지)

```
□ 영상 소스: Pexels 스톡만으로 구성하지 않았는가?
□ 인물 매칭: 언급된 인물이 화면에 나오는가?
□ 내용 매칭: 각 장면의 영상이 내레이션 내용과 직접 관련 있는가?
```
