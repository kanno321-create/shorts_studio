class Verdict(Enum):
    PASS  = "PASS"
    FAIL  = "FAIL"
    RETRY = "RETRY"

    @property
    def result(self) -> str:
        """Rubric-schema compatible string form (PASS / FAIL).

        return "PASS" if self is Verdict.PASS else "FAIL"

## Decisions Made

1. **Create `scripts/orchestrator/state.py` as a new module rather than re-export from `gate_guard`:** The Wave 0 frozen test stub imports `from scripts.orchestrator.state import Verdict` and uses `Verdict.FAIL` (enum identity). Phase 5's `gate_guard.Verdict` is a rubric-schema dataclass with `result: Literal["PASS","FAIL"]` + evidence/score — incompatible with enum identity. The two types coexist: dataclass for Inspector reports, enum for supervisor verdicts. Backward compat achieved via `Verdict.result` property.

2. **ANTHROPIC_API_KEY missing is NOT a constructor-time failure:** The Anthropic SDK is lazy — `Anthropic(max_retries=0)` succeeds without a key, authentication deferring to the first `client.messages.create()` call. This lets tests instantiate the pipeline with `mock_anthropic_client` cleanly, and production runs will surface auth errors at the first real invocation (logged, breaker-protected). `_MISSING_KEY_MSG` Korean ValueError only fires on catastrophic ImportError / runtime ctor failure, not key absence.

3. **Graceful NanoBanana + KenBurns fallback in `ShortsPipeline.__init__`:** Both adapters can legitimately fail construction (missing GOOGLE_API_KEY / missing ffmpeg binary). Wrapping in `try/except` + `logger.warning` + leaving slot `None` is Hook 3종 compatible because the except body contains an explicit `logger.warning(...)` call (NOT silent-swallow). This preserves mock-injection pathways for Phase 5/7 tests that don't set GOOGLE_API_KEY.

4. **Compact breaker factory (lambda):** 9 `CircuitBreaker(name, max_failures=3, cooldown_seconds=300)` calls compressed via `_mk_breaker = lambda n: CircuitBreaker(n, 3, 300)`. Saved ~17 lines. `# noqa: E731` acknowledged — the lambda is scoped inside `__init__` and only called 9 times. Trade-off: minor readability cost vs 800-line cap preservation.

5. **Phase 5 test rename rather than deletion:** Keeping test identity tests (3 renamed tests assert the new factory-backed defaults) preserves regression tripwire coverage. If a future refactor accidentally re-introduces NotImplementedError or changes the default class identity, these tests fail loudly.
