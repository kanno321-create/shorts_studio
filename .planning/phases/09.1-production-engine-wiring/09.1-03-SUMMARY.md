# Phase 9.1 Plan 03: Ken-Burns Adapter Summary

**One-liner:** Local FFmpeg subprocess Ken-Burns pan/zoom adapter (`KenBurnsLocalAdapter`) replacing the network-bound `ShotstackAdapter.create_ken_burns_clip` — offline, stdlib-only, Pitfall 3 anti-jitter `scale=4000:-1` prefix, Korean-first errors, with D-11 deprecation mark (body preserved) on the Shotstack method.

## Objective

Implement REQ-091-04 Wave 1 per CONTEXT D-10 (subprocess ffmpeg, no binding dep) and D-11 (keep Shotstack method, just deprecate). The ORCH-12 fallback shot path previously incurred $0.10+/call + latency + network-dependence via Shotstack; local FFmpeg removes all three while preserving Phase 7 regression behaviour (Shotstack mock in Phase 7 tests — real adapter body unchanged).

## What Changed

### 1. scripts/orchestrator/api/ken_burns.py (NEW, 192 lines)

- `KenBurnsUnavailable(RuntimeError)` exception — raised when `shutil.which("ffmpeg")` returns None at `__init__`. Message: `"ffmpeg 바이너리를 찾을 수 없습니다 — 대표님 PATH 확인 필요"`.
- `KenBurnsLocalAdapter.__init__(circuit_breaker=None, output_dir=None)` — precondition check + sane defaults (`outputs/ken_burns/`).
- `generate_clip(image, duration_seconds=5, resolution=(720,1280), fps=30, zoom_rate=0.0015) -> Path`:
  - Validates image exists → Korean `FileNotFoundError` otherwise.
  - Creates output dir on demand, filename `kb_<ms>.mp4` (millisecond-suffixed to avoid same-second collisions).
  - Builds command via pure `_build_command(...)` helper.
  - Runs via `subprocess.run(..., shell=False, timeout=120, capture_output=True, text=True)`.
  - If `circuit_breaker` supplied → wraps subprocess call via `circuit_breaker.call(lambda: subprocess.run(...))` (Phase 5 parity).
  - Non-zero rc → `RuntimeError("ffmpeg 실패 (rc=X): <stderr tail 500>")`.
- `_build_command(image, output_path, duration_seconds, resolution, fps, zoom_rate) -> list[str]` — pure staticmethod, unit-testable without subprocess:
  - Filter chain: `scale=4000:-1,zoompan=z='min(zoom+{rate},1.5)':x=iw/2-(iw/zoom/2):y=ih/2-(ih/zoom/2):d={frames}:s={w}x{h}:fps={fps}`
  - Pitfall 3 defence: `scale=4000:-1` is FIRST filter (low-res zoompan jitters without upscale).
  - Output codec: `-c:v libx264 -pix_fmt yuv420p -t {duration_seconds}`.
