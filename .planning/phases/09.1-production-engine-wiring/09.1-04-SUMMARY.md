# Phase 9.1 Plan 04: Runway Ratios + Voice Discovery Summary

**One-liner:** VALID_RATIOS_BY_MODEL per-model registry in RunwayI2VAdapter + ElevenLabsAdapter 3-tier voice_id resolution (constructor/env/API) removing the unregistered "detective_hao" default.

## Objective

Two loosely-coupled Wave 1 Track D fixes per CONTEXT D-12 (Runway model/ratio registry) and D-13 (ElevenLabs voice discovery) — both targeted at production-blocker drifts found in session #24 실측.

## What Changed

### 1. scripts/orchestrator/api/runway_i2v.py (+38 / -4)

- Added `VALID_RATIOS_BY_MODEL` module-level dict with D-12 2026-04-20 실측 values:
  - `"gen3a_turbo": ["16:9", "9:16", "768:1280", "1280:768"]`
  - `"gen4.5": ["720:1280"]`
- `DEFAULT_RATIO` now derived as `VALID_RATIOS_BY_MODEL[DEFAULT_MODEL][0]` (= `"16:9"`) so the constant and the dict cannot drift apart.
- `RunwayI2VAdapter.__init__` gains optional `model: str | None = None` and `ratio: str | None = None` kwargs. Validation logic:
  - Unknown model → `ValueError("RunwayI2VAdapter: 알 수 없는 model '{m}' — 지원 모델: [...] (대표님)")`
  - Unspecified ratio → auto-selects `valid[0]` for the chosen model
  - Invalid ratio → `ValueError("RunwayI2VAdapter: model '{m}' 의 유효 ratio 는 [...], 받은 값: '{r}' (대표님)")`
- `image_to_video(...)` kwargs dict replaces `DEFAULT_MODEL`/`DEFAULT_RATIO` with `self.model`/`self.ratio`, so swapping to `gen4.5` becomes a one-line constructor change in Phase 10.
- `__all__` exports `VALID_RATIOS_BY_MODEL`.

### 2. scripts/orchestrator/voice_discovery.py (new, +91)

- `discover_korean_default_voice(api_key: str | None = None) -> str`
- API key resolution: explicit arg → `$ELEVENLABS_API_KEY` → `$ELEVEN_API_KEY` → Korean `ValueError` on miss.
- `httpx.get(https://api.elevenlabs.io/v1/voices, headers={"xi-api-key": key, "accept": "application/json"}, timeout=30)` + `raise_for_status()` (no silent-except).
- 3-tier priority: `labels.language == "ko"` → `category == "premade"` → first voice in list.
- Empty account → `RuntimeError("대표님 ElevenLabs 계정에 사용 가능한 voice 가 없습니다 — 대시보드에서 voice 추가 또는 ELEVENLABS_DEFAULT_VOICE_ID env 설정 필요")`.
- Pure function; caching happens in the adapter (Pitfall 6).

### 3. scripts/orchestrator/api/elevenlabs.py (+43 / -5)

- New module-level constants:
  - `DEFAULT_VOICE_ID_ENV = "ELEVENLABS_DEFAULT_VOICE_ID"`
  - `_KOREAN_FALLBACK_VOICE_ID: str | None = None` (Pitfall 6 process-wide cache)
- Constructor gains `default_voice_id: str | None = None` kwarg; captures `explicit_arg or os.environ.get(DEFAULT_VOICE_ID_ENV)` snapshot into `self._default_voice_id`.
- New private method `_resolve_default_voice_id()` implements the lazy tier chain:
  1. `self._default_voice_id` (constructor/env snapshot)
  2. `_KOREAN_FALLBACK_VOICE_ID` (module cache)
  3. `discover_korean_default_voice(self._api_key)` — caches the result module-wide.
- Both `generate()` and `generate_with_timestamps()` now use `scene.get("voice_id") or self._resolve_default_voice_id()` instead of `scene.get("voice_id", "detective_hao")`. The old hardcoded literal is gone.
