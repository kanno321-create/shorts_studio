# Phase 09.1 Plan 02: Character Registry Summary

**One-liner**: Pydantic v2 CharacterEntry model (extra='forbid') + CharacterRegistry loader that resolves relative `ref_path` against `registry_path.parent` (Pitfall 5), seeded with 1 `channel_profile` entry pointing to the Wave 0 anchor PNG for Stage 2→4 smoke test.

## What Was Delivered

### Task 1: `scripts/orchestrator/character_registry.py` (99 lines)

**`CharacterEntry`** (pydantic v2 BaseModel):
- `model_config = ConfigDict(extra="forbid")` — blocks schema drift at load
- `name: str` (min_length=1, max_length=64)
- `ref_path: Path`
- `description: str` (default="", max_length=500)
- `tags: list[str]` (default_factory=list)

**`CharacterRegistry`** (stdlib + pydantic only):
- `__init__(registry_path=Path("assets/characters/registry.json"))` — records `self.assets_root = registry_path.parent`
- `load() -> self` — raises `FileNotFoundError("캐릭터 레지스트리 파일 없음: ...")` if missing; parses JSON; validates each entry; verifies ref PNG exists; returns `self` for chaining
- `get(name) -> CharacterEntry` — raises `KeyError("캐릭터 '{name}' 을 레지스트리에서 찾을 수 없음 — 사용 가능: [...]")` if absent
- `get_reference_path(name) -> Path` — calls `_resolve()` → relative paths resolve against `assets_root`, absolute paths pass through unchanged
- `list_all() -> list[str]` — sorted keys
- `_resolve(ref_path)` — private helper for Pitfall 5
