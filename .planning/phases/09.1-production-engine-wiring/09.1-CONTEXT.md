## Implementation Decisions

### producer_invoker / supervisor_invoker wiring (D-01 ~ D-03)

- **D-01: Claude Agent SDK (`anthropic` package) 직접 호출** — subprocess 기반 Claude Code CLI 호출이나 Task tool wrapping 거부. 이유: (a) production 환경이 Claude Code runtime 에 의존하지 않아야 함 (서버/crontab 실행 가능), (b) Anthropic 공식 SDK 가 가장 maintainable, (c) 토큰 사용량 직접 제어 가능.
- **D-02: Agent 정의는 `.claude/agents/producers/{name}/AGENT.md` 에서 system prompt 로드** — 기존 Phase 4 asset 재사용. Agent SDK 호출 시 `system=` 파라미터로 주입. producer-specific Tool 정의는 Phase 10 에서 추가 (Phase 9.1 은 `model.messages.create` 단순 호출만).
- **D-03: GateContext dict 를 `user` message 로 주입, JSON 응답 강제** — producer 는 `messages=[{role: "user", content: json.dumps(inputs)}]` 호출, `response_format` 또는 system prompt 에 "Return valid JSON with fields X, Y, Z" 명시. supervisor 는 producer output + gate name 을 받아 `Verdict` enum (`PASS` / `FAIL` / `RETRY`) 반환.

### Nano Banana Pro adapter (D-04 ~ D-06)

- **D-04: `scripts/orchestrator/api/nanobanana.py` 신설, google-genai SDK 사용** — shorts_naberal `_nanobanana_from_script_v3.py` 의 `build_hyperreal_prompt()` + `client.models.generate_content()` 로직 복제. API key 는 `$GOOGLE_API_KEY` (`.env` 에 이미 존재).
- **D-05: Model = `"nano-banana-pro-preview"` (default), Safety fallback 유지** — shorts_naberal 의 safer retry 로직 (SAFETY / BLOCK 키워드 감지 시 softened prompt 재호출) 그대로 포팅.
- **D-06: `NanoBananaAdapter.generate_scene(prompt: str, output_path: Path) -> Path`** — Kling/Runway 의 `image_to_video(prompt, anchor_frame)` 와 signature 대응. CircuitBreaker 지원 (Phase 5 D-6 defaults 3 failures / 300s cooldown). 응답의 `inline_data.data` 를 `output_path.write_bytes()` 로 저장.

### 캐릭터 레지스트리 (D-07 ~ D-09)

- **D-07: 파일시스템 기반 `assets/characters/` + `registry.json` manifest 하이브리드** — shorts_naberal `child/assets/characters/` 규약 상속 (`{name}_ref.png` + `{name}_real_N.png` variations). 신규 `registry.json` 은 이름 → 파일 경로 + 메타데이터(설명/tag) 매핑. DB 거부 (overkill).
- **D-08: `CharacterRegistry` util 클래스 (`scripts/orchestrator/character_registry.py`)** — 주요 메서드: `load_registry() / get_character(name) / get_reference_path(name) / list_all()`. pydantic 모델 `CharacterEntry` (name, ref_path, description, tags).
- **D-09: Phase 9.1 단계에서는 shorts_naberal 레퍼런스 파일 복사 없음** — Phase 10 에서 대표님이 주인공 캐릭터 확정 시 `assets/characters/` 채움. 9.1 은 인프라만 + 샘플 1개 (대표님이 품질 테스트에 사용하신 `profile_kr_bright.png` 를 `assets/characters/channel_profile_ref.png` 로 복사하여 smoke test 용도).

### Ken-Burns FFmpeg 로컬 (D-10 ~ D-11)

- **D-10: `scripts/orchestrator/api/ken_burns.py` 신설, subprocess ffmpeg 호출** — ffmpeg-python binding 거부 (추가 의존성). Python `subprocess.run(["ffmpeg", ...])` + `shutil.which("ffmpeg")` precondition 체크. ffmpeg 미설치 시 `KenBurnsUnavailable` exception.
- **D-11: 기존 `ShotstackAdapter.create_ken_burns_clip()` 은 deprecated, 제거 금지** — Phase 10 재설계 시 완전 제거. 9.1 에서는 `FallbackChain` 에서 새 KenBurnsLocal 을 우선 호출하도록 분기 추가만. ShotstackAdapter 는 색보정 / composite 역할로만 유지.

### RunwayI2VAdapter VALID_RATIOS_BY_MODEL (D-12)

- **D-12: 모델별 valid ratios dict 추가** — 2026-04-20 실측 기준:
  ```python
  VALID_RATIOS_BY_MODEL = {
      "gen3a_turbo": ["16:9", "9:16", "768:1280", "1280:768"],
      "gen4.5": ["720:1280"],  # 실측 확인된 값만; 추가 발견 시 확장
      # gen4_turbo, gen4, veo3, seedance2 등은 Phase 10 에서 확인
  }
  ```
  adapter constructor 에서 `ratio` 파라미터 미지정 시 model 기반 첫 항목 자동 선택. 명시 시 valid 검증.

### ElevenLabs voice_id (D-13)

- **D-13: shorts_naberal 에서 실제 운영 voice_id 추출 → adapter default 교체** — adapter 의 `"detective_hao"` default 는 대표님 계정 미등록 (2026-04-20 실측 FAIL). `shorts_naberal/scripts/` + `shorts_naberal/.env` grep 으로 실 voice_id 탐색 후 반영. 미발견 시 **대표님 계정의 default voice** 를 ElevenLabs API `GET /v1/voices` 로 조회하여 첫 번째 유효 voice_id 사용 (대화 가능한 Korean voice 우선).

### Stage 2→4 통합 smoke test (D-14 ~ D-15)

- **D-14: Smoke test scope = Nano Banana Pro scene image 1장 → Runway Gen-3a Turbo 5초 영상 1개** — 전체 orchestrator 미실행, 2-stage chain 만 검증. 목적: "(1) Nano Banana 가 shorts_studio 에서 scene 생성 가능한가, (2) 그 이미지가 Runway anchor 로 유효한가, (3) 캐릭터 consistency 가 전달되는가" 3항목.
