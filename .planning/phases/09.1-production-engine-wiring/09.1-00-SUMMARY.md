1. **RED stub pattern resolution:** Plan initially suggested `pytest.importorskip` but the correct true-RED pattern is **direct import** — ImportError surfaces as a pytest ERROR (RED), not SKIP. `importorskip` would hide the Wave 1-3 implementation gap. All 14 stubs use direct imports.

2. **test_pipeline_line_count left GREEN at Wave 0:** shorts_pipeline.py currently ~800 lines (within limit), so the invariant test PASSES today. This is intentional — it stays GREEN as a regression tripwire; it will only FAIL if Wave 2 sprawl pushes the file over 800.

3. **Windows cp949 UTF-8 guard:** Initial phase091_acceptance.py used em-dash `—` which crashed with `UnicodeEncodeError: 'cp949' codec`. Replaced with ASCII `--` and added `sys.stdout.reconfigure(encoding='utf-8')` guard (Rule 1 auto-fix, mirrors Phase 6/9 precedent).
