## Decisions Made

- **Module-level `genai` import (not lazy per-method)** ŌĆö the Wave 0 tests monkeypatch `nb_mod.genai` directly (`monkeypatch.setattr(nb_mod, "genai", MagicMock(...))`). A lazy `from google import genai` inside `_invoke_and_save` would not create the `nb_mod.genai` attribute, breaking the monkeypatch contract. Instead, import at module top wrapped in `try/except ImportError` with `genai = None` fallback, and raise explicit RuntimeError at call-time if SDK is truly absent.
- **Access `types` via `genai.types.GenerateContentConfig`** (not separate `from google.genai import types`) ŌĆö tests only patch `genai`; accessing `types` as an attribute lets the same MagicMock seamlessly stub both `Client` and `types.GenerateContentConfig`.
- **One-shot safety retry** ŌĆö matches shorts_naberal `_nanobanana_from_script_v3.py` reference exactly; prevents infinite loops on persistent policy blocks. Second failure raises `RuntimeError(...) from retry_err` preserving original cause.
- **Korean-first error surface** ŌĆö every `ValueError`/`RuntimeError`/`logger.warning` message includes ļīĆĒæ£ļŗś Ļ▓Įņ╣Ł per `CLAUDE.md` ļéśļ▓Āļ×ä Ļ░Éļ¦ł identity rules.
